Add font to a photo

Last updated on Jul 9, 2020 1 min read

在正式写推文之前，又发现了一个坑——Markdown的图片路径不支持中文。所以推文的名字不能用中文，因为推文内部的图片都是用推文名字索引的(尴尬)。

首先看下成果吧。 (原图做了压缩才push上来的)

在Y叔公众号看到一篇文章(用R给高考学生送祝福吧)，图片中添加文字，用R可以实现。觉得很有意思，于是测试了一下，竟然翻车了。所以将这个过程记录下来，以便有机会用上这个功能。

实现这个功能用的的R包并不复杂，只有showtext、ggplot2和ggimage三个。showtext用来显示个性化的字体(如本例用的maocao), ggplot2用来画图，ggimage用来添加图片背景。

一开始加载字体就出现了问题，基于某些特殊原因，未能直接访问google，所以字体只能下载后本地加载。

library(showtext)
font_add("maocao", "D:\\Data\\goupanddownwithR\\LiuJianMaoCao-Regular.ttf")

下一步按照Y叔的操作，打开showtext，创建ggplot图像，将文字添加进去，并指定了字体maocao，再讲图片导入，一切都很顺利。

然而最终出来的图和Y叔的不一样——字体没有变。

于是搜索原因，试了各种方法都没有成功。在stackoverflow上的一个回答引起了我的注意，图形设备竟然还分win和pdf以及png。

刚好这时候，查到了邱怡轩(showtext包的作者)在统计之都上的一篇文章。他用R画图，添加的文字都是用Cairo包开启的png设备。于是我也按照指引尝试了一下，果然奏效。

代码如下：

library(Cairo)
Cairo.capabilities() # 检查Cairo的兼容性
# 打开图形设备
CairoPNG("leftrighthand.jpg", 2590, 4608);
showtext_auto()

d = data.frame(x=0:1)

p = ggplot(d, aes(x,x)) + geom_blank() +
  annotate("text", x=.73, y=.91, angle = -10,
           label="左手nature\n右手science",color = "white",
           family="maocao", size=88) +
  theme_void()

#指定图片位置
img <- 'D:/Data/左手nature右手science.jpg'
ggbackground(p, img)

# 停止使用showtext
showtext_end()
# 关闭图形设备
dev.off()

最后做了个小的尝试，为了方便图片在markdown(这篇博客就是用markdown写的)中的引用，将图片上传到了新浪图床。图片上传图床用的新浪微博图床插件,在Chrome应用商店可以直接下载。登录微博账号，即可使用。拖拽上传也非常方便。

看下图床提供的图片效果，做了一下大小的适配。

One more thing.

某次看到Y叔博客的另一个包，也出现了显示问题。里面的解释让我明白了图片上加字体失败的原因，原来就是图形设备的锅。

showtext 本身和 RStudio GD 存在兼容性问题，所以在 Rstudio 中会出现各种问题，这也是为什么直接用png()保存则正常。因此给出的解决建议是即便在使用 Rstudio，也还是用系统本身的 GD 去打印，在Windows中就用 windows() 在 Mac 中就用 quartz()，然后万事大吉功成身退。
